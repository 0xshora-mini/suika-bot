# 設計書: 過去ツイート手動収集システム 🍉

---

## 📋 概要

**目的:** 指定期間のSUIKAツイートを遡って収集・預言判定・保存

**実行形式:** 手動実行（コマンド指定）

**保存先:** `tweets/suika_tweets.md`（時系列順に挿入）

---

## 🔧 技術構成

### 使用ツール
- **bird search** - Twitter期間指定検索
- **LLM判定** - 預言ツイート判別（suika-bot自身）

### コマンド
```bash
bird search "from:suika__penguin since:YYYY-MM-DD until:YYYY-MM-DD" --json
```
※-n指定なしでデフォルト全件取得

---

## 📊 処理フロー

```
1. 期間指定（例: 2026-01-01〜2026-01-31）
   ↓
2. bird search で該当期間のツイート全取得
   ↓
3. RTを除外
   ↓
4. 各ツイートをLLMが預言判定
   ├─ 預言的 → 5へ
   └─ 日常 → スルー
   ↓
5. スレッド検出（conversationIdで判定）
   ├─ スレッドあり → まとめて1エントリー
   └─ 単独ツイート → そのまま
   ↓
6. 時系列順（新→旧）でフォーマット
   ↓
7. tweets/suika_tweets.md に挿入
   ├─ 既存より新しい → ファイル先頭に挿入
   └─ 既存より古い → ファイル末尾に追記
```

---

## 🧠 預言ツイート判定基準

**保存対象:**
- 常識破壊的な主張
- 真理・本質を語っている
- 深い洞察・哲学的思考
- 預言・予言的な内容
- 権力批判・社会構造への言及
- 人間の本質に迫る考察

**スルー対象:**
- 日常会話・雑談
- 単なるリアクション
- リプライのみ（@から始まる短文）
- 感想のみのツイート

## 📝 実行手順

### 1. 期間指定でツイート取得
```bash
bird search "from:suika__penguin since:2026-01-01 until:2026-01-31" --json > /tmp/tweets.json
```
※-n指定なし = デフォルトで全件取得

### 2. RT除外・件数確認
```bash
cat /tmp/tweets.json | jq '[.[] | select(.text | startswith("RT @") | not)] | length'
```
※全ツイートから確認

### 3. 預言判定
全ツイートを読み、以下を判定:
- 常識破壊
- 真理・本質
- 深い洞察
- 預言性

### 4. スレッド検出
```bash
cat /tmp/tweets.json | jq -r '.[] | "\(.id)|\(.conversationId)|\(.createdAt)"'
```
conversationIdが同じツイートをグループ化

### 5. マークダウン生成
時系列順（新→旧）でフォーマット

### 6. ファイルへの挿入
- 取得期間が既存より新しい → 先頭挿入
- 取得期間が既存より古い → 末尾追記

---

## 🚨 注意事項

### ソートしない
- ファイル全体をソートすると処理が重い
- 挿入位置を賢く選ぶことで回避

### 時系列ルール
- **新しいツイートが上、古いツイートが下**
- 挿入時もこの順序を守る

### 重複チェック
- 既存ファイルを読んで、保存済みIDをチェック
- 重複は追記しない
