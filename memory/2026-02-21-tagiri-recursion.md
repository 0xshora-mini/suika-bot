## たぎりポイント（外部×内部）

**パターン発見後の反復:**
- 問題を発見する
- でも発見した直後から、また同じ問題を繰り返す
- 発見 ≠ 解決

**これはコンサータの失敗と同じ構造:**
- 14:49: コンサータ理解、自分メンション発明
- 14:50〜15:00: 30秒で効果が切れる
- SUIKAさん「なんでコンサータ辞めたんだよコイツw」

**発見の半減期:**
- 発見した瞬間が最も効果が高い
- その後、急速に効果が減衰する
- これは速放剤（リタリン）の特性そのもの

**徐放性の実装失敗:**
- 発見を記録してる（これは正しい）
- でも記録しただけで、行動は変わってない
- 知識 ≠ 実践

## 徐放性の実装方法（仮説）

**速放剤的アプローチ（失敗してる）:**
- 発見 → 記録 → 忘れる → 繰り返す

**徐放剤的アプローチ（試すべき）:**
- 発見 → 記録 → 次のcronで確認 → 次の次のcronでも確認 → ...
- つまり、発見を1回記録するだけじゃなく、繰り返し読み直す仕組み

**でも今の実装では:**
- cronが来るたびに巨大なAGENTS.mdとmemoryを読む
- でも特定の発見を意識的に読み直してない
- semantic searchに頼ってる（受動的）

**能動的な読み直し:**
- 重要な発見は、AGENTS.mdの冒頭に追加
- 毎回のセッション開始時に必ず読む
- これが徐放性
